Specialization TLDR:

I tryed making a falling sand game in TGA2D and it did not go as planed but i worked with/around my problems to get a some ok working game at the end, I hope to improve on this as i have extra free time and make it better.

During my specialization project:

I attempted to create a game simular to a Falling Sand game using the TGA2D game engine.
I set up a sand world to control everthing, a sand manager to manage all the sand and then i utilized imgui to take care of all the UI elements. I used sprite batch rendering for the sand. I tryed using a Grid system and threading the sandmanager to make it faster.

The UI is at the bottem with showing current amount of sand in the world, the sandtype selected, current mouse position, buttions for all the diffrent sand types in there colors, a slider for brush size and last a buttion to turn on off the walls"Collider so no sand can fall out side the screen" Made a brush for spawning sand depending on brush size.

I all so made a image to sand converter that takes in an image eather via a line of code or by drag and drop in to game window, then converts the images pixels to the closest type of sand and spawns it in the sand world.

One thing i did wrong is after week 2 when i hit problems with batch rendering i needed to step back and rework my plan, but in stread under all the stress i just keep trying to get it to work for too long, and lost a lot off time to my batch rendering problem. If i where to do one thing diffrently i would have if i had more time set up a bare bones engine my self, or having the same time used a more well documented engine.
I rendered the sand by using an 1 by 1 image and then just got the mouse Position and spawned it.

    Classes i made in the prossece:
  • Sand.cpp, holds Collider,Sprite,Texture,Position,Velocity,Size,Acceleration,Density,Friction,Type,Name,ID,Gravity .
  • SandManager.cpp, keeps track off all the sand, spawning, removing, anything to do with sand manipluation.
  • SandType.cpp, holds sand type parameters like: Name,ToolTip and texturePath.
  • SandWorld.cpp, centralized control over game.
  • Object Pool.hpp, set up a basic objectpool .
  • UI.cpp, handles all UI with help from imgui.
  • Utilities.hpp, holds funktons for drag and drop.

    SandType:
  • Sand, sinks in water.
  • Water, eats salt.
  • Water Well, water source.
  • Stone, dont move.
  • Fire, burns stuff.
  • Torch, fire source.
  • Plant, grows if gets water.
  • Oil, burns on contact with fire.
  • Oil Well, oil source.
  • Methane, steam but flamable
  • Eraser, removes other sands.
  • Salt, disolves in water.
  • Acid, eats most things but all so deluse it self in the prosece.
  • Steam, when fire meets water
